# 반도체 시료 생산주문관리 시스템 (S-Semi)

> **Claude Code 작업 가이드**: [CLAUDE.md](./CLAUDE.md)

## 프로젝트 개요

가상의 반도체 회사 **S-Semi**를 위한 콘솔 기반 시료 생산주문관리 시스템.
연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품하는 반도체 시료(Sample)의
주문·생산·출고 흐름을 체계적으로 관리한다.

## 역할 구조

| 역할 | 책임 |
|------|------|
| 고객 | 필요한 시료를 이메일로 요청 |
| 주문 담당자 | 요청에 맞게 주문서 작성 및 관리 |

---

## 주문 상태 흐름

```
RESERVED → (승인) → 재고 충분 → CONFIRMED → RELEASE
                  → 재고 부족 → PRODUCING → CONFIRMED → RELEASE
         → (거절) → REJECTED
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 (모니터링 제외) |
| `PRODUCING` | 주문 승인 완료 + 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 + 출고 대기 중 |
| `RELEASE` | 출고 완료 |

## 기능 명세

### 메인 메뉴

시스템 현황(등록 시료 수, 총 재고, 전체 주문 수, 생산라인 대기) 요약 표시.

| 메뉴 | 기능 |
|------|------|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 시료 주문 | 고객 주문 접수 (RESERVED 생성) |
| 주문 승인/거절 | RESERVED 주문에 대해 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 생산라인 조회 | 현재 생산 중인 시료 및 대기 큐 확인 |
| 출고 처리 | CONFIRMED 주문 출고 실행 → RELEASE 전환 |

---

### 시료 관리 (Sample Management)

시료는 시스템의 가장 기본 단위. **등록된 시료만 주문 가능**.

**시료 속성**

| 필드 | 설명 |
|------|------|
| 시료 ID | 고유 식별자 (예: S-001) |
| 이름 | 시료명 (예: 실리콘 웨이퍼-8인치) |
| 평균 생산시간 | 1개당 생산 소요 시간 (min/ea) |
| 수율 | 정상 시료 / 총 생산 시료 (예: 0.92) |
| 현재 재고 | 현재 보유 수량 (ea) |

**서브 메뉴**: 시료 등록 / 시료 목록 조회 (재고 포함) / 시료 검색

---

### 시료 주문 (Order Reservation)

고객이 시료를 요청하면 주문 담당자가 주문 생성.

**예약 입력 값**
- 시료 ID
- 고객명
- 주문 수량

생성 즉시 상태: `RESERVED`
주문번호 형식: `ORD-YYYYMMDD-NNNN`

---

### 주문 승인/거절 (Order Approval)

`RESERVED` 상태 주문 목록을 확인하고 처리.

**승인 시 자동 분기**

| 조건 | 처리 |
|------|------|
| 재고 ≥ 주문 수량 | 즉시 `CONFIRMED` 전환 |
| 재고 < 주문 수량 | 생산라인에 자동 등록 → `PRODUCING` 전환 |

**거절**: 즉시 `REJECTED` 전환

---

### 모니터링 (Monitoring)

**주문량 확인**: 상태별(RESERVED / CONFIRMED / PRODUCING / RELEASE) 주문 건수
- `REJECTED`는 유효한 주문이 아니므로 제외

**재고량 확인**: 시료별 현재 재고 및 주문 대비 상태 표기

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량 = 0 |

---

### 생산라인 (Production Line)

**생산 수량 계산**

```
부족분 = 주문 수량 - 현재 재고
실 생산량 = ceil(부족분 / (수율 × 0.9))
총 생산 시간 = 평균 생산시간 × 실 생산량
```

- 생산 완료 시: `PRODUCING` → `CONFIRMED` 자동 전환
- 스케줄링 전략: **FIFO** (선입선출)
- 하나의 생산 라인은 시료를 하나씩 생산 (단일 라인)

**표시 정보**
- 현재 처리 중인 주문 정보 (주문번호, 시료명, 진행률, 완료 예정 시각)
- 대기 큐 목록 (순서, 주문번호, 시료, 주문량, 부족분, 실생산량, 예상 완료)

---

### 출고 처리 (Release)

`CONFIRMED` 상태 주문에 대해 출고 실행.

- 출고 후 상태: `CONFIRMED` → `RELEASE`
- 출고 시 표기: 주문번호, 출고 수량, 처리 일시

## 개발 지침

- 모든 Repository는 **Public**으로 생성
- 시스템은 **콘솔 기반**으로 동작
- 주문이 들어온 시료에 대해서만 생산 (수요 기반 생산)
- 생산라인은 단일 라인, 시료 1개씩 순차 처리 (FIFO)
- `REJECTED` 주문은 모니터링 및 정상 흐름에서 제외
